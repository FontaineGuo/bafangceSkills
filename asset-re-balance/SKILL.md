# 资产再平衡

## 功能说明

本 skill 读取当前持仓和资产配置目标，根据用户给出的**目标总市值（绝对值）**，计算每种资产类型需要增加或减少多少金额，以使持仓比例回归目标配置。

**核心逻辑：**
- 某类资产目标市值 = 目标总市值 × 目标配置比例
- 该类资产调仓额 = 目标市值 - 当前市值（正=买入，负=卖出）

### 输入文件

- **portfolio_with_prices.csv** - 带价格的投资组合（由 fetch-asset-prices 生成）
- **asset_allocation.csv** - 资产配置目标，包含列：
  - `AssetType` - 资产类型
  - `Allocation` - 目标配置比例（百分比）
  - `Bias` - 允许偏离范围（百分比）

### 输出文件

- **re_balance.md** - Markdown 格式的调仓计划，包含调仓明细、汇总和当前持仓参考

### 参数说明

- `$0`（必填）：目标总市值，单位元，如 `100000`
- `$1`（可选）：portfolio_with_prices.csv 路径，默认 `portfolio_with_prices.csv`
- `$2`（可选）：asset_allocation.csv 路径，默认 `asset_allocation.csv`

## 执行步骤

### 1. 环境检查

1. **运行环境**
   - 使用 `uv run ${CLAUDE_SKILL_DIR}/script.py <target_total_value>` 执行脚本
   - `uv` 会根据 script.py 头部的 inline metadata 自动创建隔离虚拟环境并安装依赖
   - 如未安装 `uv`，提示用户执行 `brew install uv`

2. **检查参数**
   - 检查 `target_total_value` 是否已提供且为正数
   - 如未提供，提示用户必须指定目标总市值

3. **检查输入文件**
   - 检查 `portfolio_with_prices.csv` 是否存在，如不存在提示先运行 `fetch-asset-prices`
   - 检查 `asset_allocation.csv` 是否存在，如不存在提示创建该文件

### 2. 读取数据

1. **读取投资组合**（`portfolio_with_prices.csv`）
   - 必需列：Code, Name, AssetCategory, AssetType, MarketValue
   - 货币基金行：MarketValue 即为当前市值（TotalInvestment 仅用于成本计算，再平衡不需要）

2. **读取资产配置**（`asset_allocation.csv`）
   - 必需列：AssetType, Allocation, Bias

### 3. 计算再平衡数据

1. **计算当前总市值**：所有资产的 `MarketValue` 之和

2. **按 AssetType 分组求和**：各类型当前市值

3. **对 asset_allocation.csv 中每类资产计算：**
   - 当前市值：该类型资产 MarketValue 之和（若持仓中无该类型则为 0）
   - 当前占比：当前市值 / 当前总市值 × 100%
   - 目标市值：target_total_value × (Allocation / 100)
   - 调仓金额：目标市值 - 当前市值
   - 操作：买入（调仓 > 0）/ 卖出（调仓 < 0）/ 无需调整（调仓 = 0）

4. **处理持仓中有但 allocation 未配置的类型**：在报告末尾单独列出

### 4. 生成 re_balance.md

格式见下方「输出格式示例」。

### 5. 输出摘要

显示：
- ✓ 调仓计划生成成功
- 目标总市值 vs 当前总市值
- 需追加/可赎回金额
- 📁 文件路径
